# IntEqualsDouble
Procedures for detecting Int32 and Double comparisons under .NET Core 3.1.

背景：
这个程序做的代码很臃肿，因为我实际上只是想要快速得到结果，所以用到了多线程去吧int的最小值测试到最大值。
测试的表达式是i == double.Parse(String.Format("{0}.0", i));
也就是运算整数能否被正好与整数值的double相等。
我只知道在C标准中sqrt()对1, 4, 9这样的数字一定会返回1.0, 2.0, 3.0，
所以我就在想，C#中int和double的比较中是否会考虑这样的整数对比。

结果：
总之，测试结果就是，这样是可以的,对于所有int范围内的数，结果都是true。
！！！请注意：但是这样仅限于整数的比较，而且是在.NET Core3.1（貌似.NET Framework 4.7.1跑完前半段没有什么问题，完事没测试完毕就转到.NET Core上去测试了）下。！！！

可复用性：
这个代码，只要改一下Cal()里面的内容就可以测试别的东西。
其实，EasyFile是从别的库里面抠出来的（我自己的），也随便用，可以很方便的读写数字。

操作：
如果你关闭了控制台，它会自动保存进度到save.tmp里。如果你想修改参数，请删掉这个文件，重新来过。（设计的不好，不可以中途改参数）

性能：
在我的电脑上，用设置的参数是16,1000000，用时大概15分钟。
